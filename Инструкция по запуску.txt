- Версия Java: 17.0.4;
- Система сборки: Собирал в project structure -> Artifacts. Возможно я неправильно понял, но,
	думаю, собралось через Maven. В манифесте версия 1.0. Версия Maven 3.8.1.
- Сторонние библиотеки: нет.

Особенности:
- Файлы обязательно должны быть txt формата. 

Порядок ввода аргументов:
- Первые два это тип данных и порядок сортировки. Неважно в каком порядке их указывать: -a -i / -i -a. Программа поймет. Остальные строго по порядку: выходной файл, входные файлы. 
- Обязательно должно быть не менее двух входных файлов. 
- Если порядок сортировки в них нарушен, то сразу, как попадается число, не подходящее под порядок сортировки (например, сортировка по возрастанию, уже записано 1,2,3, а следующее число 1) - оно не печатается.

Кладем входные txt файлы в папку с MergeSorting.jar

Примеры запуска с консоли: 
java -jar MergeSorting.jar -i -d out2.txt in1.txt in2.txt in3.txt
java -jar MergeSorting.jar -a -s out.txt in1.txt in2.txt in3.txt

Особенности: строки, которые не получилось перевести в целое число, программа будет выводить в консоль.
